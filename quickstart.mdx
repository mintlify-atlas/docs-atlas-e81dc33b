This guide will walk you through setting up and running the Rock Paper Scissors game locally. You'll have both the Flask backend and React frontend running in just a few steps.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Python 3** - For running the Flask backend server
- **pip3** - Python package installer
- **Node.js and npm** - For running the React frontend
- **Git** - For cloning the repository

<Tip>
  Check your installations by running `python3 --version`, `npm --version`, and `git --version` in your terminal.
</Tip>

## Installation

<Steps>
  <Step title="Clone the Repository">
    Clone the project repository to your local machine:

    ```bash
    git clone https://github.com/carolinajimenez/rock-paper-scissors-flask-react
    ```
  </Step>

  <Step title="Navigate to Project Directory">
    Change into the project directory:

    ```bash
    cd rock-paper-scissors-flask-react
    ```
  </Step>

  <Step title="Install Backend Dependencies">
    Navigate to the Flask server directory and install Python dependencies:

    ```bash
    cd flask-server
    pip3 install -r requirements.txt
    ```

    This installs Flask 2.3.2, the only backend dependency needed.
  </Step>

  <Step title="Install Frontend Dependencies">
    In a new terminal window, navigate to the client directory and install npm packages:

    ```bash
    cd client
    npm install
    ```

    This installs React, React Icons, and all other frontend dependencies listed in `package.json`.

    <Note>
      The installation may take a few minutes as npm downloads all required packages.
    </Note>
  </Step>
</Steps>

## Running the Application

You'll need to run both the backend and frontend servers simultaneously in separate terminal windows.

<Steps>
  <Step title="Start the Flask Backend">
    In your first terminal, from the `flask-server` directory, run:

    ```bash
    python3 server.py
    ```

    You should see output indicating the Flask server is running:

    ```
    * Serving Flask app 'server'
    * Debug mode: on
    * Running on http://127.0.0.1:5000
    ```

    <Tip>
      The server runs in debug mode by default, which means it will automatically reload when you make changes to the code.
    </Tip>
  </Step>

  <Step title="Start the React Frontend">
    In your second terminal, from the `client` directory, run:

    ```bash
    npm start
    ```

    The React development server will start and automatically open your browser to `http://localhost:3000`.

    ```
    Compiled successfully!
    
    You can now view client in the browser.
    
      Local:            http://localhost:3000
    ```
  </Step>

  <Step title="Verify Both Servers are Running">
    Confirm you have:
    - Flask backend running on `http://127.0.0.1:5000`
    - React frontend running on `http://localhost:3000`

    The React app is configured to proxy API requests to the Flask server automatically.
  </Step>
</Steps>

## Making Your First Play

Now that both servers are running, let's play a game!

<Steps>
  <Step title="Open the Application">
    Navigate to `http://localhost:3000` in your web browser. You'll see the game interface with three buttons: rock, paper, and scissors.
  </Step>

  <Step title="Select Your Action">
    Click one of the three action buttons:
    - **Rock** (fist icon)
    - **Paper** (hand icon)
    - **Scissors** (peace sign icon)

    When you click a button, the `ActionButton` component sends a POST request:

    ```javascript
    fetch('/play_game', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({ action }),
    })
    ```
  </Step>

  <Step title="View the Results">
    The game instantly shows:
    - Your selected action with an animated icon
    - The computer's randomly chosen action
    - The result: "Player won!", "Player lost!", or "It's a tie"

    The selected button highlights, and the player cards update with visual styling to indicate the winner.
  </Step>

  <Step title="Play Again">
    Click any button to start a new round. Each game is independent - the computer makes a fresh random choice every time!
  </Step>
</Steps>

## Understanding the Game Flow

Here's what happens behind the scenes when you make a move:

1. **Frontend**: You click a button (e.g., rock with action value `'r'`)
2. **API Request**: React sends `{"action": "r"}` to `/play_game` endpoint
3. **Backend Logic**: Flask's `play_game()` function (`server.py:26-52`):
   - Receives your action
   - Generates a random computer action: `random.choice(['r', 'p', 's'])`
   - Calls `get_winner()` to determine the outcome
   - Returns `{"result": "...", "computer_action": "..."}`
4. **UI Update**: React updates state and displays the results

<CodeGroup>

```python server.py
@app.route("/play_game", methods=["POST"])
def play_game():
    # Retrieve user's action from the request payload
    user_action = request.json['action']
    # Generate a random action for the computer
    computer_action = random.choice(['r', 'p', 's'])

    # Determine the game result
    if user_action == computer_action:
        return {"result": "It's a tie", "computer_action": computer_action}

    if get_winner(user_action, computer_action):
        return {"result": "Player won!", "computer_action": computer_action}

    return {"result": "Player lost!", "computer_action": computer_action}
```

```python Game Logic
def get_winner(player, opponent):
    # Define winning conditions for the player
    if (player == 'r' and opponent == 's') or \
       (player == 's' and opponent == 'p') or \
       (player == 'p' and opponent == 'r'):
        return True
    return False
```

</CodeGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port 3000 or 5000 already in use">
    If you see an error about ports being in use:
    
    - For Flask (port 5000): Stop any other Flask apps or processes using port 5000
    - For React (port 3000): The development server will prompt you to use a different port (usually 3001)
    
    Alternatively, you can kill processes on these ports:
    ```bash
    # On macOS/Linux
    lsof -ti:5000 | xargs kill
    lsof -ti:3000 | xargs kill
    ```
  </Accordion>

  <Accordion title="API requests failing">
    If the game doesn't respond when you click buttons:
    
    1. Verify the Flask server is running on `http://127.0.0.1:5000`
    2. Check the browser console for error messages
    3. Ensure `package.json` has the proxy setting: `"proxy": "http://127.0.0.1:5000/"`
    4. Try restarting both servers
  </Accordion>

  <Accordion title="Module not found errors">
    If you see import errors:
    
    - **Python**: Ensure you ran `pip3 install -r requirements.txt` in the `flask-server` directory
    - **JavaScript**: Ensure you ran `npm install` in the `client` directory
    
    Try deleting `node_modules` and running `npm install` again if the issue persists.
  </Accordion>
</AccordionGroup>
