## React Component Overview

The frontend is built with React using functional components and hooks. The entire application is contained in `client/src/App.js`, which includes:

- **App**: Main container component
- **ActionButton**: Interactive button for user moves
- **Player**: Display component for player/computer state
- **ActionIcon**: Icon renderer for game actions

## Component Architecture

```
App (Main Component)
├── ActionButton (x3) - Rock, Paper, Scissors buttons
│   └── FaHandFist / FaHand / FaHandPeace icons
├── Player (x2) - Player and Computer displays
│   └── ActionIcon - Current action visualization
└── Winner Display - Game result text
```

## App Component

The main `App` component manages application state and renders the game interface.

### State Management

```jsx
// App.js:108-113
const [playerAction, setPlayerAction] = useState("start");
const [computerAction, setComputerAction] = useState("start");
const [winner, setWinner] = useState("");
const [activeButton, setActiveButton] = useState(null);
```

**State Variables**:
- `playerAction`: Current player action (`"start"`, `"r"`, `"p"`, or `"s"`)
- `computerAction`: Current computer action
- `winner`: Game result (`""`, `"Player won!"`, `"Player lost!"`, or `"It's a tie"`)
- `activeButton`: Currently active button for visual feedback

### Response Handler

```jsx
// App.js:116-120
const onActionSelected = (response, selectedAction) => {
  setPlayerAction(selectedAction);
  setComputerAction(response['computer_action'])
  setWinner(response['result'])
}
```

This callback receives the API response and updates all relevant state variables, triggering a re-render.

### JSX Structure

```jsx
// App.js:122-141
return (
  <div className="App">
```

### Click Handler with API Call

```jsx
// App.js:36-54
const handleClick = () => {
  setActiveButton(action); // Set this button as active
  // Send action to the Flask server
  fetch('/play_game', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({ action }),
  })
  .then(response => response.json())
  .then(response => {
    // Handle server response
    onActionSelected(response, action);
  })
  .catch(error => {
    console.error('Error:', error);
  });
};
```

<Steps>
  <Step title="Set Active State">
    Immediately updates the UI to show which button was clicked
  </Step>
  
  <Step title="Send API Request">
    Makes a POST request to `/play_game` with the selected action
  </Step>
  
  <Step title="Parse Response">
    Converts the JSON response from the Flask backend
  </Step>
  
  <Step title="Update Parent State">
    Calls `onActionSelected` to update the game state in the parent component
  </Step>
</Steps>

### Button Rendering

```jsx
// App.js:56-63
return (

    className={`round-btn ${action} ${activeButton === action ? 'active' : ''}`}
```

The button has dynamic classes:

- `${action}`: Action-specific class (`r`, `p`, or `s`) for colored borders
- `active`: Applied when this button is the active one (scales up)

## Player Component

The `Player` component displays the player or computer's current state.

```jsx
// App.js:87-104
function Player({name = "Player", action="start", winner}) {
  /**
   * Component representing a player in the game.
   * 
   * @param {string} name - 'Player' or 'Computer'
   * @param {string} action - Current action ('start', 'r', 'p', 's')
   * @param {string} winner - Game result
   */
  const result = (winner === undefined || winner === "" ) ? "" 
    : winner === "Player won!" ? "you" 
    : winner === "Player lost!" ? "pc" 
    : "draw";
    
  return (
    <div className={`player ${name}-${result}`}>
      <div className={`score ${name}-${result}`}>{`${name}`}</div>
      <div className="action">
          {action && <ActionIcon name={name} action={action} />}
      </div>
    </div>
  )
}
```

### Result Calculation Logic

The component determines the CSS class based on the winner:

| Winner Value | Result for Player | Result for Computer |
|--------------|-------------------|---------------------|
| `"Player won!"` | `"you"` (green border) | `"pc"` (red border) |
| `"Player lost!"` | `"pc"` (red border) | `"you"` (green border) |
| `"It's a tie"` | `"draw"` (yellow border) | `"draw"` (yellow border) |
| `""` (empty) | `""` (no special styling) | `""` (no special styling) |

<Tip>
The result string inverts for the Computer component, so when the player wins, the computer shows the "lost" styling.
</Tip>

## ActionIcon Component

The `ActionIcon` component renders the appropriate game icon (rock, paper, scissors, or start).

```jsx
// App.js:66-85
function ActionIcon({name, action}) {
  /**
   * Component for displaying action icons.
   * 
   * @param {string} name - 'Player' or 'Computer'
   * @param {string} action - 'start', 'r', 'p', or 's'
   */
  const ICONS = {
    "start": {"icon": start, "word": "start"},
    "r": {"icon": rock, "word": "rock"},
    "p": {"icon": paper, "word": "paper"},
    "s": {"icon": scissors, "word": "scissors"},
  }
  const icon = ICONS[action]["icon"]
  const action_word = ICONS[action]["word"]
  const clase = `${action_word}-${name == "Player" ? "user" : "pc" }-icon`
  
  return (
    <img src={icon} className={clase} alt="icon" />
  )
}
```
