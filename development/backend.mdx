## Flask Server Overview

The backend is a lightweight Flask application in `flask-server/server.py` that provides a single API endpoint for playing Rock Paper Scissors. The server handles:

- Receiving player actions via HTTP POST
- Generating random computer moves
- Determining game winners
- Returning game results as JSON

## API Endpoint

### POST /play_game

The sole endpoint accepts player actions and returns game results.

```python
# server.py:26-52
@app.route("/play_game", methods=["POST"])
def play_game():
    """
    Endpoint to play a game of rock, paper, scissors.

    Returns:
        dict: A dictionary containing the game result and computer's action.
            Keys:
                "result" (str): The result of the game 
                    ("It's a tie", "Player won!", or "Player lost!").
                "computer_action" (str): The computer's chosen action.
    """
    # Retrieve user's action from the request payload
    user_action = request.json['action']
    # Generate a random action for the computer
    computer_action = random.choice(['r', 'p', 's'])

    # Determine the game result based on the user's and computer's actions
    if user_action == computer_action:
        # If both actions are the same, it's a tie
        return {"result": "It's a tie", "computer_action": computer_action}

    # If the user wins, return the result indicating player victory
    if get_winner(user_action, computer_action):
        return {"result": "Player won!", "computer_action": computer_action}

    # If the user loses, return the result indicating player's defeat
    return {"result": "Player lost!", "computer_action": computer_action}
```

## Game Logic

### Winner Determination

The `get_winner()` function implements Rock Paper Scissors rules:

```python
# server.py:57-75
def get_winner(player, opponent):
    """
    Determines the winner of a rock, paper, scissors game.

    Args:
        player (str): The action chosen by the player 
            ('r' for rock, 'p' for paper, 's' for scissors).
        opponent (str): The action chosen by the opponent.

    Returns:
        bool: True if the player wins, False otherwise.
    """
    # Define winning conditions for the player
    if (player == 'r' and opponent == 's') or \
       (player == 's' and opponent == 'p') or \
       (player == 'p' and opponent == 'r'):
        return True

    # If none of the winning conditions match, the player loses
    return False
```

### Winning Conditions

| Player Choice | Computer Choice | Result | Explanation |
|---------------|-----------------|--------|-------------|
| Rock (`r`) | Scissors (`s`) | **Player wins** | Rock crushes scissors |
| Scissors (`s`) | Paper (`p`) | **Player wins** | Scissors cut paper |
| Paper (`p`) | Rock (`r`) | **Player wins** | Paper covers rock |
| Rock (`r`) | Paper (`p`) | Player loses | Paper covers rock |
| Paper (`p`) | Scissors (`s`) | Player loses | Scissors cut paper |
| Scissors (`s`) | Rock (`r`) | Player loses | Rock crushes scissors |
| Any | Same | Tie | Identical choices |

<Tip>
The function only checks winning conditions. Any other combination means the player loses (except ties, which are checked earlier in `play_game()`).
</Tip>

## Random Move Generation

The computer's move is generated using Python's `random` module:

```python
# server.py:40
computer_action = random.choice(['r', 'p', 's'])
```

**Characteristics**:
- Uniform random distribution (each option has 1/3 probability)
- Unpredictable and fair
- No strategy or pattern

<Note>
The `random.choice()` function uses Python's pseudo-random number generator, which is sufficient for a game but not cryptographically secure.
</Note>

## Game Flow Example

<Steps>
  <Step title="Receive Request">
    Flask receives POST request:
    
    ```json
    {"action": "r"}
    ```
    
    Extracts user action from `request.json['action']`
  </Step>

  <Step title="Generate Computer Move">
    Randomly selects computer action:
    
    ```python
    computer_action = random.choice(['r', 'p', 's'])
    # Result: 's' (scissors)
    ```
  </Step>

  <Step title="Check for Tie">
    Compares actions:
    
    ```python
    if user_action == computer_action:  # 'r' == 's' → False
        # Not a tie, continue...
    ```
  </Step>

  <Step title="Determine Winner">
    Calls `get_winner()`:
    
    ```python
    get_winner('r', 's')
    # Checks: player == 'r' and opponent == 's' → True
    # Player wins!
    ```
  </Step>

  <Step title="Return Result">
    Returns JSON response:
    
    ```json
    {
      "result": "Player won!",
      "computer_action": "s"
    }
    ```
  </Step>
</Steps>

- HTTP status codes (400, 500, etc.)

### Potential Improvements

```python
@app.route("/play_game", methods=["POST"])
def play_game():
    try:
        # Validate request has JSON body
        if not request.json:
            return {"error": "No JSON body provided"}, 400
        
        # Validate action parameter exists
        user_action = request.json.get('action')
        if not user_action:
            return {"error": "Missing 'action' parameter"}, 400
        
        # Validate action is valid
        if user_action not in ['r', 'p', 's']:
            return {"error": "Invalid action. Must be 'r', 'p', or 's'"}, 400
        
        # Continue with game logic...
        computer_action = random.choice(['r', 'p', 's'])
        # ...
        
    except Exception as e:
        return {"error": f"Server error: {str(e)}"}, 500
```
